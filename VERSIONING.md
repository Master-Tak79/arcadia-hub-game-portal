# VERSIONING.md

## 목적
프로젝트 버전을 일관되게 관리하기 위한 규칙입니다.

## 버전 체계
- 형식: `MAJOR.MINOR.PATCH`
- 예시: `1.4.2`
- 태그 규칙: Git 태그는 `vMAJOR.MINOR.PATCH` (예: `v1.4.2`)

## 증가 규칙 (SemVer)
- **MAJOR**: 하위 호환이 깨지는 변경
- **MINOR**: 하위 호환을 유지하는 기능 추가
- **PATCH**: 하위 호환을 유지하는 버그 수정/문서 수정

## 프리릴리스
- 필요 시 `-alpha.N`, `-beta.N`, `-rc.N` 사용
- 예시: `v1.5.0-beta.1`

## 문서 동기화 규칙
버전 관련 변경이 발생하면 아래 문서를 함께 갱신합니다.
1. `CHANGELOG.md` (사용자 관점 변경 이력)
2. `DEVLOG.md` (작업 과정/의사결정 기록)

## 릴리스 체크리스트
- [ ] 코드 변경 반영
- [ ] 테스트/검증 완료
- [ ] `CHANGELOG.md` 업데이트
- [ ] 필요한 경우 `DEVLOG.md` 업데이트
- [ ] 버전 태그 생성 (`vX.Y.Z`)
